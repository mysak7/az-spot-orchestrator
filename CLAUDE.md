## Commands

### Local Development (3 terminals)
```bash
temporal server start-dev              # Terminal 1: Temporal dev server (SQLite) + UI on :8080
uvicorn main:app --reload --port 8000  # Terminal 2: FastAPI
python worker.py                        # Terminal 3: Temporal worker
```

### Lint / Type Check
```bash
ruff check .          # lint
ruff format .         # format
mypy .                # type check
```

### Makefile Shortcuts
```bash
make setup                                                          # Generate .env from Terraform outputs + Azure CLI
make deploy                                                         # rsync repo to control-plane VM + docker compose up
make model-register NAME=llama3 TAG=llama3:8b SIZE=Standard_NC4as_T4_v3
make model-provision MODEL_ID=<id>
make model-status MODEL_ID=<id>                                     # polls every 3s
```

### CI/CD
- **CI** (non-master / PRs): `ruff check`, `ruff format --check`, `mypy`, docker build
- **CD** (master push): build + push to GHCR → deploy to control-plane VM via `az vm run-command invoke` with OIDC (no stored secrets)

## Architecture

### Big Picture
```
Client (OpenAI-compatible request)
    │
    ▼
Control-plane VM  (stable public IP)
  ├─ FastAPI :8000       — API + reverse-proxy to Spot VMs
  ├─ Temporal :7233      — workflow orchestration
  ├─ Cosmos DB           — state store (models, vm-instances, cache metadata)
  └─ ELK stack           — log aggregation (Filebeat → ES → Kibana :5601)
    │
    ▼  provision / monitor / re-provision
Spot VMs  (ephemeral, cheapest Azure region)
  └─ Ollama :11434       — serves LLM; weights on free temp disk (/mnt/resource)
```

### Request Flow
1. `POST /proxy/{model_name}/...` → `api/routes/proxy.py` looks up the single running `VMInstance` for that model in Cosmos → forwards request to `http://{vm_ip}:11434/{path}`
2. Returns 503 if no running VM; 502 if upstream error.

### Provisioning Flow
1. `POST /api/models/{id}/provision` → creates a `pending` `VMInstance` in Cosmos → starts `ProvisionVMWorkflow` in Temporal
2. Workflow: pick cheapest region (Azure Retail Prices API) → provision Spot VM with cloud-init → poll Ollama until ready → mark `running`
3. If `SkuNotAvailable`, workflow retries next-cheapest region (clean up partial resources in background, don't block)

### Eviction Recovery
1. VM polls Azure IMDS `scheduledevents` every 15 s
2. On `Preempt` event → calls `POST /api/vms/{vm_name}/evicted` on control plane
3. Control plane runs `DeleteVMWorkflow` + `ProvisionVMWorkflow` in parallel (new VM)
4. Keep-alive watchdog (60 s interval) also detects stuck instances (>20 min in non-running state) and re-provisions

### Temporal: Workflows vs Activities
- **Workflows** (`temporal/workflows/`) — deterministic; no I/O, no randomness, no DB calls
- **Activities** (`temporal/activities/`) — all side-effects: Azure SDK calls, Cosmos DB writes, Ollama polling
- Key workflows: `ProvisionVMWorkflow`, `DeleteVMWorkflow`, `SeedBlobWorkflow`, `CopyBlobWorkflow`, `SeedFilesWorkflow`

### Model Cache (avoids re-downloading weights on every new VM)
- **Blob cache** — tar + lz4-compress model weights → upload to regional Azure Blob Storage (`SeedBlobWorkflow`)
- **Files cache** — mount blob as Azure Files NFS share per region (`SeedFilesWorkflow`) — faster mount vs blob download
- Cloud-init on new VM: query `GET /api/storage/cache/source` → use blob/NFS path if available, else `ollama pull`
- Preference order: same-region blob → nearest-region blob → Ollama pull

### Key Files
| File | Purpose |
|------|---------|
| `main.py` | FastAPI app, lifespan (Cosmos + Temporal connect, inventory warm-up, keep-alive watchdog) |
| `worker.py` | Temporal worker entry point |
| `config.py` | Pydantic settings (reads `.env`) |
| `db/models.py` | Domain models: `LLMModel`, `VMInstance`, `SystemMessage`, `ModelCacheEntry`, `FilesShareEntry` |
| `temporal/types.py` | Dataclasses for workflow/activity I/O |
| `cloud_init/vm_setup.py` | Generates base64 cloud-config YAML (Ollama install, eviction monitor, model download) |
| `api/routes/proxy.py` | Reverse-proxy to Spot VM Ollama |

### Environment Variables (`.env`, generated by `make setup`)
```
COSMOS_ENDPOINT / COSMOS_KEY          — Cosmos DB (key auth, not RBAC)
AZURE_SUBSCRIPTION_ID / AZURE_RESOURCE_GROUP
AZURE_STORAGE_ACCOUNT_NAME            — Blob cache storage account
CONTROL_PLANE_URL                     — Embedded in cloud-init (must be prod public IP, not localhost)
TEMPORAL_HOST                         — localhost:7233 locally; temporal:7233 in docker-compose
AZURE_CANDIDATE_REGIONS               — JSON array; Spot VMs try regions cheapest-first
```

### Infrastructure
- Provisioned by Terraform (`terraform/`) — control-plane VM, Cosmos DB, storage accounts, NSG rules
- GitHub Actions uses OIDC (no stored credentials) — set up once with `make oidc-setup`
